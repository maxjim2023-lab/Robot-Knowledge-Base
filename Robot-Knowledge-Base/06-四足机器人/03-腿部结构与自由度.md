# 腿部结构与自由度

## 1. 一句话理解

四足机器人的腿可以先理解成“四个小机械臂”。每条腿通常有 3 个主动关节，四条腿合起来就是 12 个主动自由度。

以 Unitree Go2 为例：

```text
身体 trunk/base
  -> hip
      -> thigh
          -> calf
              -> foot
```

## 2. Go2 的四条腿

常见命名会区分前后和左右：

| 缩写 | 含义 |
|---|---|
| FL | Front Left，左前腿 |
| FR | Front Right，右前腿 |
| RL | Rear Left，左后腿 |
| RR | Rear Right，右后腿 |

每条腿可以拆成：

| 部位 | 作用直觉 |
|---|---|
| Hip | 髋部，决定腿相对身体的侧向/外展运动 |
| Thigh | 大腿，承担主要前后摆动 |
| Calf | 小腿，和大腿一起决定足端高度与前后位置 |
| Foot | 足端，与地面接触，是控制和状态估计的关键点 |

## 3. 12 自由度的直觉

| 层级 | 数量 | 说明 |
|---|---:|---|
| 每条腿关节数 | 3 | 髋部、大腿、膝部/小腿相关关节 |
| 腿数量 | 4 | 左前、右前、左后、右后 |
| 主动自由度总数 | 12 | 3 x 4 |

学习时可以先不急着推复杂动力学，而是先建立运动学直觉：

```text
关节角 q
  -> 腿部几何关系
      -> 足端位置
```

反过来，足端想放到某个位置，就需要求对应的关节角，这就是足端逆运动学的入口。

## 4. 和 ROS / URDF 的对应

| 机器人概念 | ROS / URDF 中的表达 |
|---|---|
| 身体部件 | link |
| 关节 | joint |
| 关节角 | `/joint_states` 中的 position |
| 部件坐标系 | TF frame |
| 外观 | Mesh |

在 Go2 RViz 调试中，拖动 `joint_state_publisher_gui` 的滑块，本质上是在改变某些 joint 的 position。`robot_state_publisher` 根据这些角度重新计算每个 link 的 TF，于是你能看到腿部姿态变化。

## 5. 后续学习问题

| 问题 | 下一步 |
|---|---|
| 每条腿 3 个关节分别控制足端哪个方向？ | 阅读 Go2 URDF 的 joint axis |
| 足端位置如何由 3 个关节角算出来？ | 学习足端正运动学 |
| 足端目标位置如何反求关节角？ | 学习足端逆运动学 |
| 为什么不同步态要控制四条腿的相位？ | 学习 Walk、Trot、Bound |
