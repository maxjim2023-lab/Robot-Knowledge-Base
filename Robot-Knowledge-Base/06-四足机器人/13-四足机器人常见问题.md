# 四足机器人常见问题

## 1. 为什么四足机器人通常是 12 个自由度

常见四足机器人每条腿有 3 个主动关节，四条腿共 12 个主动自由度。这样每条腿可以在三维空间中调整足端位置，满足站立、摆腿、支撑和避障的基本需求。

## 2. RViz 中 Go2 只有骨架没有外壳，说明什么

这通常不代表机器人模型完全没加载。更常见的情况是：

- URDF 和 TF 链路已经部分正常；
- RViz 能看到 link/joint 或坐标轴；
- Mesh 外观模型没有被正确找到或加载。

优先检查：

```bash
source /opt/ros/humble/setup.bash
source /root/unitree_ws/install/setup.bash
ls /root/unitree_ws/install/go2_description/share/go2_description/meshes
```

再检查 RViz 的 RobotModel Status，看是否提示某个 `.dae` 或 `package://` 路径无法解析。

## 3. RViz 和 Gazebo 中看到的机器人有什么区别

| 工具 | 机器人代表什么 |
|---|---|
| RViz | ROS 数据的可视化结果，重点是 URDF、TF、传感器和路径 |
| Gazebo | 带物理属性的仿真对象，重点是质量、惯量、碰撞、接触和控制插件 |

所以学习顺序建议是：先在 RViz 中确认结构和坐标系，再到 Gazebo 中确认动力学和控制。

## 4. 为什么四足机器人需要状态估计

四足机器人的控制器需要知道身体姿态、速度、足端是否接触地面、关节状态等信息。真实机器人中这些信息来自 IMU、编码器、足端接触估计、里程计或融合算法。

在 Go2 当前 RViz 阶段，主要观察的是模型和关节状态；进入仿真或真机控制后，状态估计会变成控制稳定性的核心。

## 5. 为什么不能只给四条腿随便摆动

四足运动不是让每条腿独立乱动。它需要协调：

- 哪些腿支撑身体；
- 哪些腿正在摆动；
- 足端落点在哪里；
- 身体重心和速度是否稳定；
- 关节力矩是否超限。

这就是步态规划、足端轨迹规划、状态估计和控制器要共同解决的问题。
