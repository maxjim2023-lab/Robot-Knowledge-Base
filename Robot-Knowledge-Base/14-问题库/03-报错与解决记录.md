## 报错记录

### Unitree Go2：RViz 只有骨架，没有 Mesh 外壳

## 1. 环境

- 系统：Windows 11 + WSL 2 + Docker
- 工具/项目：Unitree Go2 ROS2 / RViz2
- 版本：ROS 2 Humble

## 2. 报错信息

```text
RViz 中能看到坐标轴或骨架，但看不到 Go2 的外观模型。
```

## 3. 触发步骤

1. 启动 `go2_description robot.launch.py`；
2. 启动 RViz2；
3. Fixed Frame 设置为 `base_link`；
4. 添加 RobotModel；
5. 观察到 TF 或骨架存在，但没有 Go2 的 Mesh 外壳。

## 4. 原因分析

可能原因：

- 当前终端没有执行 `source /root/unitree_ws/install/setup.bash`；
- `go2_description` 的 Mesh 文件没有被安装到 `install` 目录；
- URDF 中的 `package://go2_description/...` 路径无法被解析；
- RViz RobotModel 没有正确读取 `/robot_description`；
- RViz 使用 Topic 模式异常，需要临时切换到 File 模式验证。

## 5. 解决方式

优先检查：

```bash
source /opt/ros/humble/setup.bash
source /root/unitree_ws/install/setup.bash
ls /root/unitree_ws/install/go2_description/share/go2_description/meshes
ros2 topic echo /robot_description --once
```

RViz 中检查：

- `Global Options -> Fixed Frame` 设为 `base_link`；
- 添加 `RobotModel`；
- 查看 `RobotModel -> Status`；
- 如果 Topic 模式失败，临时把 `Description Source` 改成 `File`，选择：

```text
/root/unitree_ws/install/go2_description/share/go2_description/urdf/go2_description.urdf
```

## 6. 后续避免方式

- 每个新终端都执行 ROS 2 和工作空间 source；
- 修改 URDF、Mesh、Launch 后使用 `colcon build --symlink-install`；
- 先确认 RViz 中 TF 和 RobotModel 正常，再进入 Gazebo；
- 把 Mesh 路径问题和 TF 问题分开判断：坐标轴会动不等于 Mesh 一定能显示。
