## 主题记录

### Unitree Go2 ROS2 数据流

## 1. 当前理解

当前已经知道 Go2 的最小可视化链路是：

```text
joint_state_publisher_gui -> /joint_states -> robot_state_publisher -> /tf -> RViz
```

这条链路能帮助理解 ROS 2 中模型、状态和可视化之间的关系。

## 2. 卡住的地方

- `/robot_description` 和 URDF 文件之间的关系还需要通过命令验证；
- `robot_state_publisher` 如何把 joint state 转成 TF 还需要继续理解；
- Go2 的每个 joint 名称、axis 和腿部实际运动方向还需要逐个对照。

## 3. 需要补的前置知识

- URDF 中 link / joint / axis / origin 的含义；
- TF 树和 frame 的父子关系；
- RViz RobotModel 的显示机制。

## 4. 推荐资料

- ROS 2 TF2 文档；
- ROS 2 robot_state_publisher 文档；
- `go2_description` 的 URDF / Xacro 文件；
- 当前项目记录：[[06-四足机器人/14-Unitree-Go2-ROS2项目笔记]]。

## 5. 后续行动

1. 用 `ros2 topic echo /joint_states` 记录 12 个关节名；
2. 用 RViz 逐个拖动关节滑块，观察腿部姿态；
3. 对照 URDF 中 joint axis，整理 Go2 关节表；
4. 进入 Gazebo 后继续观察同一批 topic 是否仍然存在。
