# TF 坐标变换

## 1. 一句话理解

TF 是 ROS 2 中描述“各个坐标系之间相对位置和姿态”的系统。机器人不是一个整体点，而是一串 link 和 joint 组成的坐标树；TF 让 RViz、控制算法和感知算法知道每个部件当前在哪里。

在 Unitree Go2 项目中，可以用这条链路理解 TF：

```text
joint_state_publisher_gui
  -> /joint_states
      -> robot_state_publisher
          -> /tf + /tf_static
              -> RViz
```

## 2. Go2 项目中的 TF 角色

| 对象 | 作用 |
|---|---|
| URDF / Xacro | 定义 link、joint、父子关系和静态几何关系 |
| `/joint_states` | 提供每个可动关节当前角度 |
| `robot_state_publisher` | 根据 URDF 和关节角计算每个 link 的 TF |
| `/tf_static` | 发布不随时间变化的固定坐标变换 |
| `/tf` | 发布随关节运动变化的动态坐标变换 |
| RViz | 订阅 TF 后显示机器人姿态 |

如果拖动 `joint_state_publisher_gui` 滑块后，RViz 中 Go2 的坐标轴或模型姿态发生变化，说明 `/joint_states -> robot_state_publisher -> /tf` 链路是通的。

## 3. TF 和 Mesh 的区别

| 维度 | TF | Mesh |
|---|---|---|
| 本质 | 坐标系之间的数学变换 | 机器人外观模型 |
| 决定什么 | 每个 link 在哪里、朝向哪里 | 看起来像什么 |
| 常见话题/文件 | `/tf`、`/tf_static`、URDF joint | `.dae`、`.stl` |
| 调试现象 | 坐标轴会动 | 外壳是否显示 |

所以“只有骨架没有外壳”通常说明 TF 可能已经正常，但 RViz 没有加载到 Mesh。

## 4. 常用检查命令

```bash
ros2 topic list | grep tf
ros2 topic echo /tf
ros2 topic echo /tf_static --once
ros2 topic echo /joint_states
```

查看机器人描述是否发布：

```bash
ros2 topic echo /robot_description --once
```

## 5. Go2 调试中的判断标准

| 现象 | 说明 |
|---|---|
| RViz Fixed Frame 报 `map` 不存在 | 当前没有发布 `map`，可改为 `base_link` |
| Fixed Frame 改为 `base_link` 后不报错 | TF 树里存在 `base_link` |
| 拖动滑块后坐标轴变化 | `/joint_states` 和 `/tf` 链路正常 |
| 坐标轴变化但没有外壳 | Mesh 路径、安装或 RViz RobotModel 配置有问题 |
| RobotModel 报红 | 检查 `robot_description`、URDF 和 Mesh 路径 |
