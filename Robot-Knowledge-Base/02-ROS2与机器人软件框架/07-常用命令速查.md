# ROS2 常用命令速查

## 1. 环境加载

```bash
source /opt/ros/humble/setup.bash
source /root/unitree_ws/install/setup.bash
```

每个新终端都要重新 source。否则当前 shell 不知道 ROS 2 命令、工作空间包和 `package://` 资源路径。

## 2. 工作空间编译

```bash
cd /root/unitree_ws
colcon build --symlink-install
source install/setup.bash
```

只编译 Go2 模型包：

```bash
colcon build --packages-select go2_description --symlink-install
```

## 3. 包、节点、话题

```bash
ros2 pkg list | grep -E "go2|unitree"
ros2 node list
ros2 topic list
ros2 topic info /joint_states
ros2 topic echo /joint_states
ros2 topic echo /tf
ros2 topic echo /robot_description --once
```

## 4. Unitree Go2 RViz 联调

启动状态发布器：

```bash
ros2 launch go2_description robot.launch.py
```

启动关节滑块：

```bash
ros2 run joint_state_publisher_gui joint_state_publisher_gui
```

启动 RViz：

```bash
export MESA_D3D12_DEFAULT_ADAPTER_NAME=NVIDIA
export LC_NUMERIC=C
rviz2
```

## 5. Mesh 排查

```bash
ls /root/unitree_ws/install/go2_description/share/go2_description/meshes
ls /root/unitree_ws/install/go2_description/share/go2_description/urdf
```

如果 RViz 只显示骨架：

1. 确认已经 source 工作空间；
2. 确认 install 目录中存在 Mesh；
3. 查看 RViz RobotModel 的 Status；
4. 临时用 File 模式手动选择 URDF。

## 6. ROS 2 daemon

当 topic/node 显示异常时，可重启 daemon：

```bash
ros2 daemon stop
ros2 daemon start
```

## 7. Docker / WSL

```bash
wsl -d Ubuntu
docker start unitree_lab
docker exec -it unitree_lab /bin/bash
wsl --shutdown
```
