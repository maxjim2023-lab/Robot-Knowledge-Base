# Obsidian 使用说明

## 1. Obsidian 是什么

Obsidian 是一款基于本地 Markdown 文件的知识库管理工具。它的核心思想是：把知识拆成一篇篇笔记，再通过双向链接、标签、目录和图谱把它们组织起来。

它适合用来管理：

- 学习笔记
- 项目文档
- 技术知识库
- 读书笔记
- 问题记录
- 复盘与经验总结

当前知识库已经采用了比较清晰的主题式结构，例如：

- `Robot-Knowledge-Base/00-总览与路线图`
- `Robot-Knowledge-Base/01-机器人基础`
- `Robot-Knowledge-Base/02-ROS2与机器人软件框架`
- `Robot-Knowledge-Base/03-仿真软件`
- `Robot-Knowledge-Base/04-AMR移动机器人`
- `Robot-Knowledge-Base/05-机械臂`
- `Robot-Knowledge-Base/06-四足机器人`
- `Robot-Knowledge-Base/07-双足与人形机器人`
- `Robot-Knowledge-Base/08-感知与智能控制`
- `Robot-Knowledge-Base/09-控制与强化学习`

建议继续围绕这套结构维护，不要频繁大规模调整目录。

## 2. 基本概念

### 2.1 库

库是 Obsidian 管理笔记的根目录。当前文件夹就是一个 Obsidian 库。

一个库本质上就是一组普通文件和文件夹，即使不使用 Obsidian，也可以用任意文本编辑器打开。

### 2.2 笔记

Obsidian 中的笔记通常是 `.md` 文件。Markdown 是一种轻量标记语言，适合记录结构化文本。

常用写法：

```markdown
# 一级标题
## 二级标题
### 三级标题

- 无序列表
- 另一项

1. 有序列表
2. 第二项

**加粗**
`行内代码`
```

### 2.3 双向链接

双向链接是 Obsidian 的核心功能。使用 `[[笔记名称]]` 可以链接到另一篇笔记。

示例：

```markdown
[[01-机器人系统组成]]
[[ROS2总体理解]]
[[PID与PD控制]]
```

建议在笔记中主动链接相关概念，而不是只依赖目录查找。

### 2.4 标签

标签用于跨目录聚合内容。

示例：

```markdown
#机器人
#ROS2
#控制
#仿真
#待整理
#问题记录
```

建议标签少而稳定，不要为每篇笔记随意创建大量近义标签。

## 3. 推荐的知识库组织方式

### 3.1 用目录管理领域

目录适合表达较稳定的大分类。例如当前机器人知识库可以保持如下逻辑：

```text
Robot-Knowledge-Base/
  00-总览与路线图/
  01-机器人基础/
  02-ROS2与机器人软件框架/
  03-仿真软件/
  04-AMR移动机器人/
  05-机械臂/
  06-四足机器人/
  07-双足与人形机器人/
  08-感知与智能控制/
  09-控制与强化学习/
```

目录的作用是让知识库有稳定骨架。

### 3.2 用链接管理关系

链接适合表达知识之间的真实关系。

例如在 `PID与PD控制` 中可以链接：

```markdown
相关内容：

- [[控制系统总览]]
- [[关节PD与阻抗控制]]
- [[阻抗控制]]
- [[MPC模型预测控制]]
```

这样可以把不同目录下的相关知识串起来。

### 3.3 用索引页管理入口

每个大目录建议有一个入口页，用来说明这个主题下有哪些内容、推荐阅读顺序是什么。

例如：

- `00-机器人学习总览.md`
- `01-两个月学习路线.md`
- `03-核心概念索引.md`
- `04-开源项目索引.md`

推荐每个主题目录都保持一个“总览页”或“索引页”。

示例：

```markdown
# ROS2 与机器人软件框架总览

## 推荐阅读顺序

1. [[ROS2总体理解]]
2. [[Node-Topic-Service-Action]]
3. [[Launch文件]]
4. [[TF坐标变换]]
5. [[RViz可视化]]
6. [[ros2_control控制框架]]

## 常用速查

- [[常用命令速查]]
- [[常见报错与解决]]
```

## 4. 笔记命名规范

### 4.1 推荐命名方式

建议继续使用当前这种方式：

```text
01-主题名称.md
02-主题名称.md
03-主题名称.md
```

优点：

- 顺序清楚
- 适合系统学习
- 文件列表可读性强
- 方便后续补充和调整

### 4.2 命名建议

文件名尽量做到：

- 简短
- 明确
- 可搜索
- 避免过度抽象

推荐：

```text
06-SLAM建图.md
07-定位Localization.md
08-路径规划.md
```

不推荐：

```text
一些想法.md
学习记录.md
资料整理.md
新建文档.md
```

## 5. 单篇笔记推荐结构

建议每篇知识类笔记采用固定结构。

模板：

```markdown
# 标题

## 1. 一句话理解

用一两句话说明这个概念是什么。

## 2. 核心概念

- 概念 A：
- 概念 B：
- 概念 C：

## 3. 工作原理

说明它为什么这样工作，以及关键流程。

## 4. 典型应用

- 应用场景 1
- 应用场景 2
- 应用场景 3

## 5. 常见问题

### 问题 1

回答。

### 问题 2

回答。

## 6. 相关笔记

- [[相关笔记1]]
- [[相关笔记2]]
- [[相关笔记3]]
```

对于项目笔记，可以使用：

```markdown
# 项目名称

## 1. 项目目标

## 2. 环境配置

## 3. 运行步骤

## 4. 关键代码或命令

## 5. 遇到的问题

## 6. 解决方案

## 7. 后续改进
```

## 6. 常用 Markdown 写法

### 6.1 标题

```markdown
# 一级标题
## 二级标题
### 三级标题
```

建议一篇笔记只有一个一级标题。

### 6.2 列表

```markdown
- 第一项
- 第二项
- 第三项
```

```markdown
1. 第一步
2. 第二步
3. 第三步
```

### 6.3 任务清单

```markdown
- [ ] 待完成
- [x] 已完成
```

适合用于问题清单、学习计划、项目待办。

### 6.4 代码块

使用三个反引号包裹代码。

````markdown
```bash
ros2 topic list
ros2 node list
ros2 launch package_name launch_file.py
```
````

### 6.5 表格

```markdown
| 名称 | 作用 | 备注 |
|---|---|---|
| Topic | 异步消息通信 | 适合传感器数据 |
| Service | 请求-响应通信 | 适合查询和设置 |
| Action | 长时间任务 | 适合导航和机械臂动作 |
```

### 6.6 引用

```markdown
> 这是一段引用内容。
```

适合记录书籍、论文、文档中的关键观点。

## 7. Obsidian 常用功能

### 7.1 快速切换

使用快速切换可以通过标题或文件名快速打开笔记。

建议习惯使用搜索，而不是只从目录树点击。

### 7.2 全局搜索

全局搜索适合查找某个概念在整个知识库中出现的位置。

例如可以搜索：

- `SLAM`
- `ros2_control`
- `MPC`
- `足端轨迹`
- `坐标变换`

### 7.3 反向链接

反向链接可以显示哪些笔记引用了当前笔记。

适合用来发现一个概念在知识库中的上下文，例如：

- 哪些笔记提到了 `TF坐标变换`
- 哪些笔记引用了 `状态估计`
- 哪些笔记和 `强化学习基础` 有关

### 7.4 图谱视图

图谱视图可以查看笔记之间的链接关系。

它适合发现知识结构，但不建议为了图谱好看而强行添加无意义链接。链接应该服务于理解。

### 7.5 模板

如果经常创建同类笔记，建议使用模板。

可以建立一个目录：

```text
Templates/
```

然后放入：

```text
知识笔记模板.md
项目笔记模板.md
问题记录模板.md
读书笔记模板.md
```

## 8. 推荐插件

### 8.1 核心插件

建议开启：

- 文件列表
- 搜索
- 反向链接
- 图谱视图
- 命令面板
- 模板
- 日记

### 8.2 社区插件

可按需安装：

- Dataview：用于查询和汇总笔记
- Calendar：配合日记使用
- Templater：增强模板能力
- Excalidraw：绘制结构图、流程图、手绘图
- Advanced Tables：增强表格编辑
- Omnisearch：增强全文搜索

插件不宜一次安装太多。先用核心功能建立稳定工作流，再逐步添加。

## 9. 推荐工作流

### 9.1 学习新主题

步骤：

1. 在对应目录下创建新笔记
2. 写下“一句话理解”
3. 记录核心概念
4. 添加关键公式、图示或命令
5. 链接相关笔记
6. 回到索引页补充入口

示例：

```markdown
# ZMP与支撑多边形

## 1. 一句话理解

ZMP 是判断双足机器人动态平衡的重要概念。

## 6. 相关笔记

- [[质心-CoM]]
- [[足底接触与平衡]]
- [[双足机器人总体架构]]
```

### 9.2 记录项目问题

遇到问题时，不建议只把解决方案塞进聊天记录或临时文档。可以建立专门的问题笔记。

模板：

```markdown
# 问题：标题

## 1. 现象

## 2. 环境

## 3. 报错信息

## 4. 排查过程

## 5. 最终解决方案

## 6. 经验总结

## 7. 相关笔记
```

适合放在：

```text
Robot-Knowledge-Base/02-ROS2与机器人软件框架/08-常见报错与解决.md
```

或者单独建立：

```text
Robot-Knowledge-Base/10-问题与经验库/
```

### 9.3 每周整理

建议每周做一次轻量整理。

检查内容：

- 是否有孤立笔记
- 是否有重复主题
- 是否有笔记缺少相关链接
- 是否有临时笔记需要归档
- 是否有问题已经解决但未总结

每周整理不要变成大扫除，重点是让知识继续可用。

推荐控制在少量稳定标签。

示例：

```markdown

#机械臂
#移动机器人
#四足机器人
```

这种标签很有价值，因为它们表达的是笔记状态。

## 11. 知识库维护原则

### 11.1 先记录，再整理

学习和项目推进时，先把信息记下来，不要一开始就追求完美结构。

可以先使用：

```markdown
#待整理
```

后续再归档到合适目录。

### 11.2 一篇笔记解决一个主题

不要把太多无关内容塞进同一篇笔记。

推荐：

- `TF坐标变换.md`
- `RViz可视化.md`
- `ros2_control控制框架.md`

不推荐：

- `ROS2学习.md`
- `机器人所有知识.md`

### 11.3 经常补链接

一篇笔记写完后，至少补充 2 到 5 个相关链接。

这会让知识库从“文件夹集合”逐渐变成“知识网络”。

### 11.4 少折腾工具，多积累内容

Obsidian 的价值不在插件数量，而在长期稳定的记录、链接和复盘。

建议优先维护：

- 清楚的目录
- 稳定的命名
- 可读的笔记结构
- 高质量的相关链接
- 定期复盘

## 13. 每日使用建议

每天使用 Obsidian 时，可以遵循一个简单流程：

1. 先打开总览或路线图
2. 进入今天要学习的主题
3. 新增或补充相关笔记
4. 把新笔记链接到已有笔记
5. 用任务清单记录未解决问题
6. 学习结束前补一句总结

示例：

```markdown
## 今日总结

今天理解了 ROS2 中 Topic、Service、Action 的区别。下一步需要结合 TurtleBot3 项目看实际使用方式。

## 待继续

- [ ] 阅读 [[Launch文件]]
- [ ] 运行 TurtleBot3 仿真
- [ ] 整理常见 ROS2 命令
```

## 14. 快捷键建议

可以根据个人习惯设置快捷键。常用功能建议绑定：

| 功能 | 用途 |
|---|---|
| 快速切换 | 快速打开笔记 |
| 全局搜索 | 查找知识点 |
| 命令面板 | 执行 Obsidian 命令 |
| 插入模板 | 快速创建结构化笔记 |
| 打开图谱 | 查看知识关系 |
| 切换阅读/编辑模式 | 阅读和编辑切换 |

快捷键不需要一次设置很多，先设置最常用的几个即可。

## 15. 总结

Obsidian 的最佳用法不是把所有资料都堆进去，而是持续把知识变成可搜索、可链接、可复用的笔记。

建议长期坚持三件事：

1. 每个主题都有清晰笔记
2. 每篇笔记都有相关链接
3. 每周做一次轻量整理

对于当前机器人知识库，最重要的是围绕已有目录继续积累内容，同时通过索引页和双向链接把机器人基础、ROS2、仿真、控制、机械臂、移动机器人、四足和双足机器人这些主题连接起来。
